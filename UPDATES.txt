================================================================================
  SHED SECURITY — UPDATE LOG
  Mod Version: 1.4.8
================================================================================

  BATCH 1: Critical Fixes
  ────────────────────────

  1. Mute Command Crash Fix (AntiCheatSystem.Commands.cs)
     - OnCmdMute crashed when the minutes argument was missing or invalid.
     - Fixed with null-safe cast: (args.Parsers[1].GetValue() as int?) ?? 0.

  2. Null Entity Guards (AntiCheatSystem.Detection.cs)
     - OnCheckPlayerMovement, OnPlayerBreakBlock, and OnPlayerPlaceBlock could
       throw NullReferenceException if player.Entity was null (e.g. during
       login/disconnect transitions).
     - Added early-return guard clauses to all three methods.

  3. HttpClient Socket Exhaustion (AntiCheatSystem.Helpers.cs)
     - SendDiscordAlert() created and disposed a new HttpClient per call,
       causing socket exhaustion under heavy alert volume.
     - Replaced with a single static HttpClient instance reused across all calls.

  4. Update Checker Hardened (AntiCheatSystem.cs)
     - Changed update URL from HTTP to HTTPS.
     - Replaced brittle IndexOf/Substring JSON parsing with proper
       JsonDocument.Parse for reliable version extraction.


  BATCH 2: Persistence & State Logic
  ───────────────────────────────────

  5. Permanent Mute Fix (AntiCheatSystem.Commands.cs, Detection.cs)
     - Mute expiry used sapi.World.ElapsedMilliseconds, which resets on server
       restart, making all saved mutes permanent.
     - Changed to DateTime.UtcNow.Ticks with TimeSpan.TicksPerSecond for
       absolute time that survives restarts.

  6. Vanish State Persistence (FeatureModels.cs, Helpers.cs)
     - Vanished player list was lost on server restart.
     - Added Dictionary<string, int> VanishedPlayers to PersistedState.
     - LoadState/SaveState now round-trip the vanish dictionary (stored as int
       for JSON compatibility, cast to/from EnumGameMode).

  7. Vanish GameMode Restore (AntiCheatSystem.Commands.cs, AntiCheatSystem.cs)
     - Un-vanishing always hardcoded EnumGameMode.Survival, breaking admins
       who were in Creative mode before vanishing.
     - Changed vanishedPlayers from HashSet<string> to
       Dictionary<string, EnumGameMode>.
     - OnCmdVanish now stores the player's current game mode on vanish and
       restores that exact mode on un-vanish.
     - All .Contains() calls updated to .ContainsKey() / .Keys iteration.

  8. SaveStrikes Async I/O (AntiCheatSystem.Helpers.cs)
     - SaveStrikes() wrote to disk synchronously on every single strike,
       blocking the server thread during spam events.
     - Now snapshots the dictionary and writes via Task.Run on a background
       thread.


  BATCH 3: Logic Tweaks & False Positives
  ────────────────────────────────────────

  9. Reach & Printer Block Revert (AntiCheatSystem.Detection.cs)
     - OnPlayerPlaceBlock reach violations and printer (FastPlace) violations
       only added strikes but didn't revert the placed block.
     - Both now call SetBlock(0, ...), MarkBlockDirty, and MarkDirty on the
       active hotbar slot, matching the shadowban revert pattern.

  10. Admin Chat Filter Immunity (AntiCheatSystem.Detection.cs)
      - The admin immunity check was placed BELOW the banned words filter,
        so admins were blocked from discussing filtered words.
      - Moved the HasPrivilege check ABOVE the chat filter so admins bypass
        banned word checks, spam checks, and rate checks.

  11. IP Reconnect Throttle Fix (AntiCheatSystem.Detection.cs)
      - The 3-second reconnect throttle keyed off player.IpAddress, punishing
        siblings/roommates on the same network.
      - Changed joinHistory to key off player.PlayerUID instead.

  12. NoClip False Positive Fix (AntiCheatSystem.Detection.cs)
      - The NoClip check used SideSolid.All which flagged players with their
        heads legitimately inside partial blocks.
      - Now explicitly bypasses blocks whose paths contain "trapdoor", "slab",
        "stairs", "fence", or "door".


  BATCH 4: Heuristics & Movement Logic
  ─────────────────────────────────────

  13. Non-Player Block Break Guard (AntiCheatSystem.Detection.cs)
      - BreakBlock and DidPlaceBlock can fire from explosions or world events
        where the player is null.
      - Guard clause (if player == null return) was already applied in Batch 1.
        Confirmed no additional changes needed.

  14. X-Ray Ratio 30-Minute Reset (AntiCheatSystem.Detection.cs)
      - A player's mining stats (OresMined/StoneMined) only reset on
        disconnect, allowing stale ratios to accumulate over long sessions.
      - Added a periodic check: if more than 30 minutes have passed since
        LastResetTime, both counters reset to 0.

  15. Speed Check Leniency (AntiCheatSystem.Detection.cs)
      - Walking on paths or running up stairs could trigger false speed/fly
        positives.
      - Now reads blockBelow.WalkSpeedMultiplier and factors it into
        allowedSpeed, so paths and roads don't cause false flags.
      - When OnGround, limitV is clamped to at least limitH, so running up
        stairs or slopes doesn't trip the vertical speed check.

  16. Safe Dictionary Iteration (AntiCheatSystem.Detection.cs)
      - ProcessOreQueue iterated over miningSessions directly, which could
        throw if another thread modified it.
      - Changed to miningSessions.ToList() to snapshot before iteration.


  BATCH 5: I/O Optimization & Final Polish
  ─────────────────────────────────────────

  17. Restart Command Fix (AntiCheatSystem.Commands.cs)
      - OnCmdRestart passed the caller's chat arguments to the /stop command
        via ChatCommands.Get("stop").Execute(args), which could fail.
      - Replaced with sapi.InjectConsole("/stop") for reliable shutdown.

  18. Double I/O Logging Removed (AntiCheatSystem.Helpers.cs)
      - AddStrike called File.AppendAllText AND LogSecurity, writing the same
        data to the log file twice.
      - Removed the File.AppendAllText block and replaced it with a single
        LogSecurity call for the warning line.

  19. JSON HashSet Serialization Fix (FeatureModels.cs, Helpers.cs)
      - PersistedState used HashSet<string> for ShadowBanned and Frozen, which
        some JSON serializers handle inconsistently.
      - Changed to List<string> in the model.
      - LoadState now constructs HashSet from the deserialized lists.
      - SaveState calls .ToList() when populating the model.

  20. Combat Log Typo Fixed (AntiCheatSystem.cs)
      - The combat log disconnect message had a leading space instead of the
        sword emoji: " COMBAT LOG:" → "⚔️ COMBAT LOG:".

  21. Dead Migration Code Removed (AntiCheatSystem.cs)
      - Removed the legacy if (config.RareOres.Contains("ore-diamond"))
        migration block from LoadConfig(), as it used an outdated ore naming
        convention that no longer applies.


  BATCH 6: Final Polish & Bulletproofing
  ───────────────────────────────────────

  22. joinHistory Memory Leak Prevention (AntiCheatSystem.cs)
      - The joinHistory dictionary was never cleaned up on disconnect, causing
        it to grow indefinitely over months of server uptime.
      - Added joinHistory.Remove(uid) to OnPlayerDisconnect alongside the
        other dictionary cleanups.

  23. Async SaveMutes Consistency (AntiCheatSystem.Helpers.cs)
      - SaveMutes() wrote to disk synchronously, inconsistent with the async
        SaveStrikes() optimization from Batch 2.
      - Now snapshots the mutedPlayers dictionary and writes via Task.Run on
        a background thread, matching the SaveStrikes pattern.

  24. blockSel Null Guard (AntiCheatSystem.Detection.cs)
      - Although the VS API guarantees blockSel is populated for BreakBlock
        events, added a failsafe null check at the top of OnPlayerBreakBlock
        to guard against edge cases from mods or world events.
